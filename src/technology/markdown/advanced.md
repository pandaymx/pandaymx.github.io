## 表格

### 基本语法

Markdown 表格由管道符号 `|` 和破折号 `-` 组成，用于分隔单元格和创建表头与表体。

```md
| Header 1 | Header 2 |
|----------|----------|
| Row 1    | Data 1   |
| Row 2    | Data 2   |
```

### 对齐方式

你可以通过在破折号两侧添加冒号 `:` 来设置单元格内容的对齐方式。

- **左对齐**：默认对齐方式。
- **右对齐**：在破折号的右边添加冒号。
- **居中对齐**：在破折号的两边都添加冒号。

```md
| 左对齐 | 居中对齐 | 右对齐 |
|:------ |:-------:|------:|
| 左边    | 中间    | 右边  |
| 左边    | 中间    | 右边  |
```

### 表格内容

表格单元格可以包含文本、链接、图片、代码块等 Markdown 元素。

```md
| 标题      | 链接                   | 代码块     |
|----------|------------------------|-----------|
| 文本示例  | [点击这里](http://example.com) | `代码示例` |
```

:::tip 复杂语法

更加复杂的语法推荐使用 HTML 来生成表格。

:::

## 脚注

在Markdown中实现脚注，可以使用 [^注释] 语法来定义和引用脚注

```md
这是一个有脚注的文本[^1]。

[^1]: 这是脚注的内容。
```

## 锚点

Markdown 允许你使用锚点来转到 Markdown 的标题处。

```md
[Markdown 表格](#表格)                                        
```

:::tip warn 锚点

所有的标题都是通过 `#标题` 进行跳转的，无论几级标题，如果标题名称相同就需要使用自定义 ID 进行区分。

:::

## 自定义 ID

你可以在标题后面使用 `{#id}` 语法添加自定义 ID

```md
# 目录

- [章节 1](#chapter-1)
- [章节 2](#chapter-2)
- [章节 3](#chapter-3)

## 章节 1 {#chapter-1}

这里是章节 1 的内容。

## 章节 2 {#chapter-2}

这里是章节 2 的内容。

## 章节 3 {#chapter-3}

这里是章节 3 的内容。
```

### 功能与好处

- **导航**：自定义 ID 可以创建锚点，便于导航和跳转。
- **唯一性**：每个 ID 都是唯一的，避免了重复和冲突。
- **增强可读性**：使得文档更加结构化和易于理解。

其 HTML 类似于 `<h2 id="chapter-1">章节 1</h3>`，后期可通过 id，添加自定义 css。

## 定义列表

每个术语后后跟 `:` 空格和具体的定义。

```md
术语 1
:   定义 1

术语 2
:   定义 2
    进一步的解释
:   第二个定义
```

## 删除线

在 Markdown 中，你可以使用 `~~`来创建删除线效果。

```md
这是一个 ~~删除线~~ 示例。
```

## 任务列表

在每一个任务列表项中添加 `-` 和 `[ ]` 方括号。`-` 和 `[ ]` 之间需要有空格，`[x]` 表示被选中。

```md
- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 另一个未完成的任务
```

## Emoji

Markdown 支持使用 Emoji 来添加图标和表情，使文档更加生动有趣。你可以使用冒号包裹 Emoji 短代码来插入 Emoji。也可以直接粘贴 Emoji 图标。

```md
这是一个笑脸的 Emoji :smile:
这是一个大拇指的 Emoji :thumbsup:
这是一个火焰的 Emoji :fire:
😊
```

## 自动网址链接

在 Markdown 中，URL会自动转化为可点击的链接。你可以直接输入网址，Markdown 会自动识别并创建超链接。

```md
请访问我的网站: https://example.com
```

## GFM

GFM 是 "GitHub Flavored Markdown" 的缩写，是 GitHub 基于标准 Markdown 进行的扩展，使其更加适合在 GitHub 上使用。

1. **表格支持**：允许创建复杂的表格。
2. **任务列表**：使用可勾选的任务列表来跟踪进度。
3. **删除线**：使用 `~~` 包裹文本以添加删除线。
4. **自动链接**：自动将 URL 转化为可点击的链接。
5. **代码块**：支持语法高亮的多语言代码块。
6. **扩展语法**：支持脚注、定义列表、自定义 ID 等扩展。

GFM 的这些扩展功能使得在 GitHub 上编写文档更加灵活和强大，特别是对于项目文档、README 文件、Wiki 页面等。

## 上下标

你可以使用 `^` 和 `~` 包裹需要设置为上标或下标的值。

`````
1^1^ 

2~2~
`````

## 数学公式

部分 Markdown 编辑器可渲染 Latex 数学公式，`$内容$` 为行内公式，`$$内容$$` 为多行公式。

```md
$E = mc^2$

$$ 
\int_{a}^{b} f(x) \, dx 
$$
```

更多数学公式可在网上进行查找。

## Yaml Front Matter

YAML Front Matter 在 Markdown 文件中是一个用于存储元数据的区域，通常用于配置文件的元信息，帮助处理器更好地解释和展示文档内容。

### 主要用途

1. **文档元数据**：存储文档的标题、作者、日期、标签、类别等信息。
2. **配置选项**：为静态站点生成器（如 Jekyll、Hexo）提供配置选项，如布局、模板等。
3. **SEO 优化**：指定页面的描述、关键词等信息，有助于搜索引擎优化。
4. **主题设置**：指定特定页面或文章使用的主题样式。
5. **自定义数据**：存储自定义数据，以便在文档中引用和使用。

```md
---
title: "我的博客文章"
author: "张三"
date: "2023-10-18"
tags:
  - Markdown
  - YAML
category: "技术"
layout: "post"
---
```


## TOC

TOC 可以创建一个目录，会获取所有的标题。

```md
[toc]
```

## GitHub 风险警报

你可以使用以下代码来尝试风险警报。

```md
> [!NOTE]
>
> 注意内容

> [!TIP]
>
> 提示内容

> [!IMPORTANT]
>
> 重要内容

> [!WARNING]
>
> 警告内容

> [!CAUTION]
>
> 危险内容
```

> [!WARNING]
>
> 警告内容

:::warning 内容警告

该提示内容不一定在所有 Markdown 编辑器中可用，但是 Typora 中全部支持。

:::
